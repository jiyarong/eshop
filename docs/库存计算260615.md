一 账面库存

公式：
  账面库存 = 总入库数量 − 净销售数量 − 总损毁/损耗

其中净销售数量 = 销售订单数量 − 反向订单数量（退货）；损毁/损耗暂跳过。

账面库存是跨平台汇总的，一个 SKU 在 WB 和 Ozon 上的销售合并计算。


#### 采购数量

来源：ec_sku_batches（SKU 采购批次表）
取值：状态为 received 或 closed 且 batch_type = normal 的批次，SUM(received_quantity)
说明：与平台无关，所有平台共用同一批采购数据；仅统计正常入库批次。


#### 补正调整数量

来源：ec_sku_batches（SKU 采购批次表）
取值：状态为 received 或 closed 且 batch_type != normal 的批次，SUM(received_quantity)
说明：通常为补正、损耗或其他修正，可能为负数。


#### 总入库数量

公式：总入库数量 = 采购数量 + 补正调整数量


#### WB 销售数量（全量订单）

来源：ec_order_items
取值：SUM(quantity) WHERE platform='wb' AND platform_sku_id IN wb_nm_ids AND 非取消
说明：按 ec_sku_products 绑定后的 WB 全量订单统计，不再限制 fulfillment_type='fbs'。


#### WB 退货

来源：raw_wb_goods_returns
取值：COUNT WHERE nm_id IN wb_nm_ids
说明：WB 的 return 记录不需要关联订单，直接按 nm_id 统计即可。
      如果仍沿用 completed_dt 口径，则只统计 completed_dt 有值的记录。


#### Ozon 销售数量

来源：ec_orders + ec_order_items（已聚合自 Ozon 原始数据）
取值：排除已取消订单后，SUM(ec_order_items.quantity) WHERE platform='ozon'
注意：ec_order_items.sku_code 在部分行为 NULL（导入时 ec_sku_products 未录入映射），
      因此必须通过 ec_sku_products 取 ozon_sku（ozon_product_id → raw_ozon_products.raw_json['sku']）过滤。


#### Ozon 退货

来源：raw_ozon_returns
逻辑：ec_order_items JOIN raw_ozon_returns ON order_number + ozon_sku 双条件精确匹配


#### 损毁/损耗

暂跳过，后续补充。


#### 汇总公式（按 SKU）

账面库存 =
  总入库数量
  − (WB净额 + Ozon净销售)

其中：
  总入库数量 = 采购数量 + 补正调整数量
  采购数量   = SUM(received_quantity WHERE batch_type = normal)
  补正调整数量 = SUM(received_quantity WHERE batch_type != normal)

  WB净额   = WB销售 − WB退货
  WB销售   = SUM(ec_order_items.quantity) WHERE platform='wb' AND platform_sku_id IN nm_ids AND 非取消
  WB退货   = COUNT(raw_wb_goods_returns WHERE nm_id IN nm_ids)

  Ozon净销售 = Ozon销售 − Ozon退货
  Ozon销售   = SUM(ec_order_items.quantity) WHERE platform='ozon' AND platform_sku_id IN (ozon_skus) AND 订单未取消
  Ozon退货   = SUM(raw_ozon_returns.quantity)，通过 order_number + ozon_sku JOIN 到 ec_order_items

说明：
  各平台的 supply_quantity、WB FBW 送仓量等数据不再参与账面库存数量计算，
  仅可作为补充信息或后续展示数据使用。


Ruby 代码示例：

```ruby
sku_code = "KJ-228-BK"

# 采购数量：normal 批次
purchase_quantity = Ec::SkuBatch
  .where(sku_code: sku_code, status: %w[received closed], batch_type: :normal)
  .sum(:received_quantity)

# 补正调整数量：非 normal 批次
adjustment_quantity = Ec::SkuBatch
  .where(sku_code: sku_code, status: %w[received closed])
  .where.not(batch_type: Ec::SkuBatch.batch_types[:normal])
  .sum(:received_quantity)

received_quantity = purchase_quantity + adjustment_quantity

# 通过 ec_sku_products 取平台原生 ID
wb_nm_ids = Ec::SkuProduct
  .where(sku_code: sku_code, platform: "wb")
  .pluck(:product_id)

ozon_product_ids = Ec::SkuProduct.where(sku_code: sku_code, platform: "ozon").pluck(:product_id)
ozon_skus = ozon_product_ids.empty? ? [] :
  RawOzon::Product.where(ozon_product_id: ozon_product_ids)
                  .pluck(Arel.sql("raw_json->>'sku'")).compact

# Ozon 销售：按 platform_sku_id 过滤，非取消订单
not_cancelled = ->(rel) { rel.joins(:order).where.not(ec_orders: { order_status: "cancelled" }) }
ozon_sold = ozon_skus.empty? ? 0 :
  not_cancelled.(Ec::OrderItem.where(platform: "ozon", platform_sku_id: ozon_skus)).sum(:quantity)

# Ozon 退货：SUM raw_ozon_returns.quantity，通过 order_number + ozon_sku JOIN
ozon_returns = ozon_skus.empty? ? 0 :
  RawOzon::Return
    .joins(<<~SQL)
      JOIN ec_orders
        ON ec_orders.external_order_number = raw_ozon_returns.order_number
       AND ec_orders.platform = 'ozon'
      JOIN ec_order_items
        ON ec_order_items.order_id = ec_orders.id
       AND ec_order_items.platform_sku_id::bigint = raw_ozon_returns.ozon_sku
    SQL
    .where(ec_order_items: { platform_sku_id: ozon_skus })
    .where.not(ec_orders: { order_status: "cancelled" })
    .sum("raw_ozon_returns.quantity")

# WB: 全量订单（不再限制 FBS）
wb_sales = wb_nm_ids.empty? ? 0 :
  not_cancelled.(Ec::OrderItem.where(platform: "wb", platform_sku_id: wb_nm_ids)).sum(:quantity)

# WB 退货：直接按 nm_id 统计，不关联订单
wb_goods_return = wb_nm_ids.empty? ? 0 :
  RawWb::GoodsReturn.where(nm_id: wb_nm_ids).count

wb_net     = wb_sales - wb_goods_return
net_sales  = wb_net + ozon_sold - ozon_returns
book_stock = received_quantity - net_sales

puts <<~RESULT
  SKU:           #{sku_code}
  采购数量:      #{purchase_quantity}
  补正调整数量:   #{adjustment_quantity}
  总入库:        #{received_quantity}
  WB 销售:       #{wb_sales}
  WB 退货:       #{wb_goods_return}
  WB 净额:       #{wb_net}
  Ozon 销售:     #{ozon_sold}
  Ozon 退货:     #{ozon_returns}
  净销售:        #{net_sales}
  账面库存:      #{book_stock}
RESULT
```


二 白俄可用库存

白俄可用库存是对账面库存分布的描述。账面库存算出"还有多少件未售出"，但这些货散在不同位置——俄罗斯平台仓、卖家 FBS 仓（白俄仓中已锁定待发的部分）、正在运往平台途中（暂跳过）。去掉这些后，剩下的就是白俄仓里可以自由调配的库存。

公式：
  白俄可用 = 账面库存
            − WB 在库（FBW）
            − WB FBS 在库
            − Ozon 在库（FBO）
            − Ozon FBS 在库

所有在库数字均实时调平台 API 获取当前快照，不走 DB 缓存。


### ec_sku_products —— 平台商品 ID 的权威来源

ec_sku_products 是 sku_code 与各店铺平台商品的硬关联表，查平台在库时统一从这里取 product_id，不再做 vendor_code 模糊匹配。

| 字段 | 说明 |
|---|---|
| sku_code | 内部 SKU 编码 |
| store_id | 关联 ec_stores（含 platform、wb_raw_account_id、ozon_raw_account_id） |
| platform | 'wb' / 'ozon' |
| product_id | WB: nm_id；Ozon: ozon_product_id |
| offer_id | Ozon: offer_id（article），WB 为空 |
| platform_sku_id | WB: chrt_id（尺码 ID），Ozon 为空 |

注意：唯一键为 (store_id, product_id)，不含 sku_code。因此同一 SKU 在同一店铺可能对应多个 product_id（例如同一商品在平台上有多条 nm_id）——查询时必须取所有匹配行，累加库存。


---

### 各平台接口说明

#### WB 在库数量（FBW）

接口：3 步异步报告（seller-analytics-api 域名）

  Step 1  GET /api/v1/warehouse_remains?groupByNm=true
          触发生成库存报告任务，返回 { "data": { "taskId": "..." } }

  Step 2  GET /api/v1/warehouse_remains/tasks/{task_id}/status
          轮询直到 data.status = "done"（通常数秒到数十秒）

  Step 3  GET /api/v1/warehouse_remains/tasks/{task_id}/download
          返回 JSON 数组，每行含 nmId 和 quantity

nm_id 来源：ec_sku_products.product_id（platform='wb'，对应店铺的 wb_raw_account_id）
过滤方式：report 行中 nmId 在 nm_ids 集合内的累加 quantity

注意：每个账号只需触发一次报告，全量下载后按 nm_id 过滤，无需逐 nm_id 分别请求。


#### WB FBS 在库数量

接口：
  Step 1  GET /api/v3/warehouses（marketplace-api）
          获取卖家自仓列表，筛选 deliveryType=1 的 FBS 仓库

  Step 2  POST /api/v3/stocks/{warehouseId}（marketplace-api）
          参数：{ "chrtIds": [chrt_id, ...] }
          返回：当前各 chrtId 的 FBS 可发货数量（stocks[].amount）

chrt_id 来源：ec_sku_products.platform_sku_id（platform='wb'，对应店铺的 wb_raw_account_id）
注意：platform_sku_id 为空的行（该商品暂未录入 chrt_id）跳过。


#### Ozon 在库数量（FBO + FBS）

接口：
  POST /v4/product/info/stocks（api-seller.ozon.ru，cursor 翻页）
  参数：{ filter: {}, limit: 100 }
  返回：每个商品的 stocks 数组，包含 fbo 和 fbs 两种类型

取值：
  FBO 在库 = stocks[type='fbo'].present
  FBS 在库 = stocks[type='fbs'].present

product_id 来源：ec_sku_products.product_id（platform='ozon'，对应店铺的 ozon_raw_account_id）
过滤方式：response 行中 product_id 在 ozon_product_ids 集合内的累加（替代 offer_id 字符串比对）

### 示例：以 SKU KJ-228-BK 计算白俄可用库存

同一个 SKU 会在多个店铺上架，需遍历所有 ec_stores 累加后才能得到完整的平台在库数。
一个 SKU 在同一店铺可能有多个 product_id，全部累加。

账面库存（来自第一节计算）= 185 件

实时调 API 逐店铺获取快照，各店铺累加：

  WB TaxiLink（store_id=X, wb_raw_account_id=2）：
    FBW 在库 = 45 件，FBS 在库 = 8 件

  WB WorldChoice（store_id=Y, wb_raw_account_id=3）：
    FBW 在库 = 12 件，FBS 在库 = 2 件

  Ozon Nevastal（store_id=Z, ozon_raw_account_id=1）：
    FBO 在库 = 30 件，FBS 在库 = 3 件

  Ozon Nevastal2（store_id=..., ozon_raw_account_id=3）：
    FBO 在库 = 0 件，FBS 在库 = 0 件

  Ozon Domos（store_id=..., ozon_raw_account_id=4）：
    FBO 在库 = 5 件，FBS 在库 = 1 件

  Ozon Nanokit（store_id=..., ozon_raw_account_id=5）：
    FBO 在库 = 0 件，FBS 在库 = 0 件

计算：
  WB 合计   = (45+12) FBW + (8+2) FBS = 67 件
  Ozon 合计 = (30+0+5+0) FBO + (3+0+1+0) FBS = 39 件
  白俄可用  = 185 − 67 − 39 = 79 件


Ruby 代码示例：

```ruby
sku_code = "KJ-228-BK"

# 账面库存（来自第一节逻辑）
book_stock = calculate_book_stock(sku_code)  # => 185


# 从 ec_sku_products 加载该 SKU 所有店铺的平台商品映射
# 一个 SKU 在同一店铺可能有多个 product_id（全部需要）
sku_products = Ec::SkuProduct
  .where(sku_code: sku_code)
  .joins(:store)
  .includes(:store)

wb_fbw_stock = 0
wb_fbs_stock = 0

# === WB：按账号分组，逐账号调一次 API ===
wb_by_account = sku_products
  .select { |sp| sp.store.wb? }
  .group_by { |sp| sp.store.wb_raw_account_id }

wb_by_account.each do |raw_account_id, sps|
  account    = RawWb::SellerAccount.find(raw_account_id)
  wb_client  = RawWb::WbClient.new(account.api_token)

  nm_ids   = sps.map(&:product_id).uniq              # WB: product_id = nm_id
  chrt_ids = sps.map(&:platform_sku_id).compact.uniq # WB: platform_sku_id = chrt_id

  # FBW 在库：3 步异步报告（Step 1 是 GET，带 query 参数）
  task    = wb_client.get(:seller_analytics, '/api/v1/warehouse_remains', groupByNm: true)
  task_id = task['data']['taskId']
  loop do
    status = wb_client.get(:seller_analytics, "/api/v1/warehouse_remains/tasks/#{task_id}/status")
    break if status['data']['status'] == 'done'
    sleep 3
  end
  report = wb_client.get(:seller_analytics, "/api/v1/warehouse_remains/tasks/#{task_id}/download")
  wb_fbw_stock += Array(report)
    .select { |row| nm_ids.include?(row['nmId'].to_s) }
    .sum { |row| row['quantity'].to_i }

  # FBS 在库：用 chrt_id 查卖家自仓库存
  next if chrt_ids.empty?
  fbs_warehouses = wb_client.get(:marketplace, '/api/v3/warehouses')
    .select { |w| w['deliveryType'] == 1 }
  fbs_warehouses.each do |wh|
    resp = wb_client.post(:marketplace, "/api/v3/stocks/#{wh['id']}", { chrtIds: chrt_ids })
    wb_fbs_stock += (resp['stocks'] || []).sum { |s| s['amount'].to_i }
  end
end

ozon_fbo_stock = 0
ozon_fbs_stock = 0

# === Ozon：按账号分组，逐账号调一次 API ===
ozon_by_account = sku_products
  .select { |sp| sp.store.ozon? }
  .group_by { |sp| sp.store.ozon_raw_account_id }

ozon_by_account.each do |raw_account_id, sps|
  account      = RawOzon::SellerAccount.find(raw_account_id)
  ozon_client  = RawOzon::OzonClient.new(account)
  product_ids  = sps.map(&:product_id).uniq  # Ozon: product_id = ozon_product_id

  cursor = nil
  loop do
    body = { filter: {}, limit: 100 }
    body[:cursor] = cursor if cursor
    resp  = ozon_client.post('/v4/product/info/stocks', body)
    items = resp['items'] || []

    items.each do |item|
      next unless product_ids.include?(item['product_id'].to_s)
      stocks = Array(item['stocks'])
      ozon_fbo_stock += (stocks.find { |s| s['type'] == 'fbo' } || {})['present'].to_i
      ozon_fbs_stock += (stocks.find { |s| s['type'] == 'fbs' } || {})['present'].to_i
    end

    cursor = resp['cursor']
    break if cursor.blank? || items.size < 100
  end
end

# === 白俄可用 ===
blr_available = book_stock - wb_fbw_stock - wb_fbs_stock - ozon_fbo_stock - ozon_fbs_stock

puts <<~RESULT
  SKU:           #{sku_code}
  账面库存:      #{book_stock}
  WB FBW 在库:   #{wb_fbw_stock}
  WB FBS 在库:   #{wb_fbs_stock}
  Ozon FBO 在库: #{ozon_fbo_stock}
  Ozon FBS 在库: #{ozon_fbs_stock}
  白俄可用:      #{blr_available}
RESULT
```

```ruby
# 以 NEVASTAL（account_id=1）为例，Minsk 时区
from_dt = Time.find_zone('Minsk').parse('2025-01-01 00:00:00')
to_dt   = Time.find_zone('Minsk').parse('2026-05-31 23:59:59')

# FBO
fbo = RawOzon::PostingItem
  .where(account_id: 1, posting_type: 'fbo')
  .where(posting_number: RawOzon::PostingFbo
    .where(account_id: 1, in_process_at: from_dt..to_dt)
    .where.not(status: 'cancelled')
    .select(:posting_number))
  .group(:offer_id).sum(:quantity)

# FBS
fbs = RawOzon::PostingItem
  .where(account_id: 1, posting_type: 'fbs')
  .where(posting_number: RawOzon::PostingFbs
    .where(account_id: 1, in_process_at: from_dt..to_dt)
    .where.not(status: 'cancelled')
    .select(:posting_number))
  .group(:offer_id).sum(:quantity)

# 合并（offer_id 即 Article，与 Ozon 报告的商品列对应）
combined = fbo.merge(fbs) { |_, a, b| a + b }
```

- 报告按 Article（offer_id）汇总，DB 侧也用 `offer_id` 分组对比
- 时间维度：报告筛选的是订单进入处理的日期，与 DB 的 `in_process_at` 对应
- 报告不含取消单，DB 侧需排除 `status='cancelled'` 的 posting
