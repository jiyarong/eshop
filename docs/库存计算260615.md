库存计算260615


一 账面库存

公式：
  账面库存 = 总采购数量 − 净销售数量 − 总损毁/损耗

其中净销售数量 = 销售订单数量 − 反向订单数量（退货）；损毁/损耗暂跳过。

账面库存是跨平台汇总的，一个 SKU 在 WB 和 Ozon 上的销售合并计算。


#### 采购数量

来源：ec_sku_batches（SKU 采购批次表）
取值：状态为 received 或 closed 的批次，SUM(received_quantity)
说明：与平台无关，所有平台共用同一批采购数据。


#### 销售数量

来源：ec_orders + ec_order_items（已聚合自各平台原始数据）
取值：排除已取消订单后，SUM(ec_order_items.quantity)
范围：WB 和 Ozon 的 FBO、FBS 订单统一计入，不区分平台。


#### 反向订单（退货）

退货订单需从账面销售中扣除：

Ozon：
  来源：raw_ozon_returns
  逻辑：ec_order_items JOIN raw_ozon_returns，能 JOIN 上的订单行不计入净销售

WB：
  来源：raw_wb_stats_sales
  逻辑：ec_order_items JOIN raw_wb_stats_sales WHERE sale_id LIKE 'R%'，能 JOIN 上的不计入净销售
  说明：WB stats_sales 中 sale_id 以 'R' 开头的记录代表退货冲销。


#### 损毁/损耗

暂跳过，后续补充。


#### 汇总公式（按 SKU）

账面库存 =
  SUM(ec_sku_batches.received_quantity WHERE status IN ('received', 'closed'))
  − SUM(ec_order_items.quantity
        WHERE 订单未取消
        AND 未关联退货记录（Ozon: raw_ozon_returns；WB: raw_wb_stats_sales sale_id LIKE 'R%'）
       )


二 白俄可用库存

白俄可用库存是对账面库存分布的描述。账面库存算出"还有多少件未售出"，但这些货散在不同位置——俄罗斯平台仓、卖家 FBS 仓（白俄仓中已锁定 FBS 发货的部分）、正在运往平台途中（暂跳过）。去掉这些后，剩下的就是白俄仓里可以自由调配的库存。

公式：
  白俄可用 = 账面库存
            − WB 在库（FBW）
            − WB FBS 在库
            − Ozon 在库（FBO）
            − Ozon FBS 在库

所有数字均从平台 API 拉快照，存入 raw_* 表，计算时只读 DB。


---

### 各平台数据来源

#### WB 在库数量（FBW）

接口：3 步异步报告（statistics-api 域名）

  Step 1  POST /api/v1/warehouse_remains
          触发创建库存报告任务，返回 task_id

  Step 2  GET  /api/v1/warehouse_remains/tasks/{task_id}/status
          轮询，直到 status = done

  Step 3  GET  /api/v1/warehouse_remains/tasks/{task_id}/download
          下载 ZIP，解压得到 Excel，按 nmId 汇总在库数量

说明：/api/v1/supplier/stocks 已标记为 Deprecated，新库存快照统一走上述异步报告接口。


#### WB FBS 在库数量

接口：
  Step 1  GET  /api/v3/warehouses（marketplace-api）
          获取卖家自仓列表，筛选 deliveryType=1 的 FBS 仓库

  Step 2  POST /api/v3/stocks/{warehouseId}（marketplace-api）
          参数：{skus: [barcode, ...]}
          返回：当前 FBS 可发货数量（amount）

存入：raw_wb_fbs_stocks.amount，按 barcode → nm_id 关联


#### Ozon 在库数量（FBO）

接口：
  POST /v4/product/info/stocks（cursor 翻页）
  返回：stocks[{type: 'fbo', present, reserved}]

存入：raw_ozon_product_stocks.present_fbo


#### Ozon FBS 在库数量

同一接口返回：stocks[{type: 'fbs', present, reserved}]

存入：raw_ozon_product_stocks.present_fbs

### 示例：以 SKU KJ-228-BK 计算白俄可用库存

假设某日快照数据：

  账面库存（来自第一节计算）= 185 件

  WB TaxiLink（account_id=2）：
    FBW 在库  = 45 件  ← warehouse_remains 报告，按 nmId 汇总
    FBS 在库  = 8 件   ← raw_wb_fbs_stocks.amount，按 nm_id 汇总

  Ozon Nevastal（account_id=1）：
    FBO 在库  = 30 件  ← raw_ozon_product_stocks.present_fbo
    FBS 在库  = 3 件   ← raw_ozon_product_stocks.present_fbs

计算：
  白俄可用 = 185 − 45 − 8 − 30 − 3 = 99 件


Ruby 代码示例：

```ruby
sku_code = "KJ-228-BK"

# 账面库存（来自第一节逻辑，此处假设已算好）
book_stock = calculate_book_stock(sku_code)  # => 185

# WB：通过 barcode → nm_id 关联汇总
nm_ids = RawWb::Product.where("vendor_code ILIKE ?", sku_code).pluck(:nm_id)

wb_fbw_stock = RawWb::Stock
  .joins("JOIN raw_wb_product_skus ps ON ps.barcode = raw_wb_stocks.barcode")
  .joins("JOIN raw_wb_products p ON p.id = ps.product_id")
  .where(p: { nm_id: nm_ids })
  .sum(:quantity)
# 注意：raw_wb_stocks 是旧接口数据，新逻辑改从 warehouse_remains 报告写入新表

wb_fbs_stock = RawWb::FbsStock
  .where(nm_id: nm_ids)
  .sum(:amount)

# Ozon：通过 offer_id 关联汇总
ozon_product_ids = RawOzon::Product
  .where(offer_id: sku_code)
  .pluck(:ozon_product_id)

ozon_fbo_stock = RawOzon::ProductStock
  .where(ozon_product_id: ozon_product_ids)
  .sum(:present_fbo)

ozon_fbs_stock = RawOzon::ProductStock
  .where(ozon_product_id: ozon_product_ids)
  .sum(:present_fbs)

# 白俄可用
blr_available = book_stock - wb_fbw_stock - wb_fbs_stock - ozon_fbo_stock - ozon_fbs_stock

puts <<~RESULT
  SKU:       #{sku_code}
  账面库存:  #{book_stock}
  WB FBW在库: #{wb_fbw_stock}
  WB FBS在库: #{wb_fbs_stock}
  Ozon FBO在库: #{ozon_fbo_stock}
  Ozon FBS在库: #{ozon_fbs_stock}
  白俄可用:  #{blr_available}
RESULT
```

注：WB FBW 在库的最终数据源待 warehouse_remains 报告同步落库后，
    替换为新表的查询，mapper 逻辑不变（仍按 nm_id 汇总）。
