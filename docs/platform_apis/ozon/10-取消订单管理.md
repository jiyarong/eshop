# 10 取消订单管理

> **模块说明**：管理 rFBS 订单的条件取消申请——查询待审批的取消申请、批准或拒绝取消请求。

共 **3** 个接口。

---

## API 列表

| # | 接口名称 | HTTP 方法 |
|---|---------|----------|
| 1 | [获取 rFBS 取消申请列表](#1-获取-rfbs-取消申请列表) | `POST /v2/conditional-cancellation/list` |
| 2 | [确认 rFBS 取消申请](#2-确认-rfbs-取消申请) | `POST /v2/conditional-cancellation/approve` |
| 3 | [拒绝 rFBS 取消申请](#3-拒绝-rfbs-取消申请) | `POST /v2/conditional-cancellation/reject` |

---

## 1. 获取 rFBS 取消申请列表

```
POST /v2/conditional-cancellation/list
```

分页获取 rFBS 订单的取消申请列表，可按发起人、货件号、状态筛选。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `limit` | integer | ✅ | 每页数量，最大 500 |
| `last_id` | integer | | 分页游标：上一页最后一条记录的 ID |
| `filters` | object | | 筛选条件（见下表） |
| `with.counter` | boolean | | 是否返回 `ON_APPROVAL` 状态的计数 |

**filters 子字段：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `cancellation_initiator` | array[string] | 取消发起方：`OZON`、`SELLER`、`CLIENT`、`SYSTEM`、`DELIVERY` |
| `posting_number` | array[string] | 货件编号列表 |
| `state` | string | 申请状态：`ALL` / `ON_APPROVAL` / `APPROVED` / `REJECTED` |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result` | array[object] | 取消申请详情列表 |
| `result[].cancellation_id` | integer | 取消申请 ID |
| `result[].posting_number` | string | 货件编号 |
| `result[].cancellation_initiator` | string | 取消发起方 |
| `result[].cancellation_reason.id` | integer | 取消原因 ID |
| `result[].cancellation_reason.name` | string | 取消原因名称 |
| `result[].cancellation_reason_message` | string | 发起方填写的备注 |
| `result[].state.id` | integer | 状态 ID |
| `result[].state.state` | string | 状态：`ALL` / `ON_APPROVAL` / `APPROVED` / `REJECTED` |
| `result[].cancelled_at` | string (datetime) | 申请创建时间 |
| `result[].order_date` | string (datetime) | 订单创建时间 |
| `result[].approve_comment` | string | 审批备注 |
| `result[].approve_date` | string (datetime) | 审批日期 |
| `result[].auto_approve_date` | string (datetime) | 自动审批截止日期（到期自动确认） |
| `result[].tpl_integration_type` | string | 物流集成类型 |
| `counter` | integer | `ON_APPROVAL` 状态申请计数 |
| `last_id` | integer | 分页游标 |

### 请求示例

```http
POST /v2/conditional-cancellation/list
Content-Type: application/json

{
  "filters": {
    "cancellation_initiator": ["CLIENT"],
    "posting_number": ["34009011-0094-1"],
    "state": "ALL"
  },
  "limit": 500,
  "last_id": 0,
  "with": { "counter": true }
}
```

### 响应示例

```json
{
  "result": [
    {
      "cancellation_id": 283784254,
      "posting_number": "34009011-0094-1",
      "cancellation_initiator": "CLIENT",
      "cancellation_reason": {
        "id": 1,
        "name": "买家改变主意"
      },
      "cancellation_reason_message": "不需要了",
      "state": { "id": 1, "name": "待审批", "state": "ON_APPROVAL" },
      "cancelled_at": "2024-11-27T12:31:43Z",
      "order_date": "2024-11-27T12:31:43Z",
      "auto_approve_date": "2024-11-28T12:31:43Z",
      "tpl_integration_type": "ozon"
    }
  ],
  "counter": "1",
  "last_id": 283784254
}
```

## 2. 确认 rFBS 取消申请

```
POST /v2/conditional-cancellation/approve
```

批准状态为 `ON_APPROVAL` 的取消申请。批准后订单将被取消，款项退还给买家。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `cancellation_id` | integer | ✅ | 取消申请 ID |
| `comment` | string | | 备注说明 |

### 响应

- **200**: 申请已确认（无特殊响应体）
- **default**: 错误信息

### 请求示例

```http
POST /v2/conditional-cancellation/approve
Content-Type: application/json

{
  "cancellation_id": 283784254,
  "comment": "同意取消"
}
```

### 错误响应示例

```json
{
  "code": 403,
  "message": "申请状态不允许此操作",
  "details": []
}
```

## 3. 拒绝 rFBS 取消申请

```
POST /v2/conditional-cancellation/reject
```

拒绝状态为 `ON_APPROVAL` 的取消申请。拒绝后订单保持当前状态，需继续向买家发货。请在 `comment` 中说明拒绝原因。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `cancellation_id` | integer | ✅ | 取消申请 ID |
| `comment` | string | | 拒绝原因（建议填写） |

### 响应

- **200**: 申请已拒绝
- **default**: 错误信息

### 请求示例

```http
POST /v2/conditional-cancellation/reject
Content-Type: application/json

{
  "cancellation_id": 283784254,
  "comment": "商品已发出，无法取消"
}
```
