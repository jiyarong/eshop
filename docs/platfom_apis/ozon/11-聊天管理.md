# 11 聊天管理

> **模块说明**：管理买家和卖家之间的聊天会话——获取聊天列表、消息历史、发送消息、标记已读等。

共 **6** 个接口。

---

## API 列表

| # | 接口名称 | HTTP 方法 |
|---|---------|----------|
| 1 | [获取聊天列表](#1-获取聊天列表) | `POST /v1/chat/list` |
| 2 | [获取聊天历史](#2-获取聊天历史) | `POST /v1/chat/history` |
| 3 | [发送消息](#3-发送消息) | `POST /v1/chat/send/message` |
| 4 | [获取未读消息数](#4-获取未读消息数) | `POST /v1/chat/unread` |
| 5 | [标记消息已读](#5-标记消息已读) | `POST /v1/chat/mark/read` |
| 6 | [更新聊天状态](#6-更新聊天状态) | `POST /v1/chat/update/status` |

---

## 1. 获取聊天列表

```
POST /v1/chat/list
```

获取卖家的所有聊天会话列表，支持按未读/已关闭等状态筛选和分页。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `filter` | object | ✅ | 筛选条件 |
| `limit` | integer | ✅ | 每页数量 |
| `last_id` | string | | 分页游标 |

**filter 子字段：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `unread_only` | boolean | 只获取有未读消息的聊天 |
| `closed_only` | boolean | 只获取已关闭的聊天 |
| `date_from` | string (date) | 起始日期 |
| `date_to` | string (date) | 截止日期 |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result` | array[object] | 聊天列表 |
| `result[].chat_id` | string | 聊天 ID |
| `result[].chat_type` | string | 聊天类型（如 `order`） |
| `result[].opponent.name` | string | 买家名称 |
| `result[].opponent.type` | string | 对手类型（`buyer`） |
| `result[].last_message.text` | string | 最后一条消息文本 |
| `result[].last_message.created_at` | string (datetime) | 最后消息时间 |
| `result[].unread_count` | integer | 未读消息数 |
| `result[].is_closed` | boolean | 是否已关闭 |

### 请求示例

```http
POST /v1/chat/list
Content-Type: application/json

{
  "filter": { "unread_only": false },
  "limit": 50
}
```

## 2. 获取聊天历史

```
POST /v1/chat/history
```

获取指定聊天会话的完整消息历史，支持按消息 ID 分页（滚动加载更早的消息）。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `chat_id` | string | ✅ | 聊天 ID |
| `from_message_id` | string | | 分页：从哪条消息开始加载更早的 |
| `limit` | integer | ✅ | 每页数量，最大 100 |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result` | array[object] | 消息列表（按时间正序） |
| `result[].id` | string | 消息 ID |
| `result[].text` | string | 消息文本内容 |
| `result[].created_at` | string (datetime) | 发送时间 |
| `result[].user.is_seller` | boolean | 发送方是否为卖家 |
| `result[].file_info` | object | 附件信息（如有） |
| `has_next` | boolean | 是否有更早的消息 |

### 请求示例

```http
POST /v1/chat/history
Content-Type: application/json

{
  "chat_id": "ch_abc123",
  "limit": 50
}
```

## 3. 发送消息

```
POST /v1/chat/send/message
```

向指定聊天会话发送文本消息，支持附带文件。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `chat_id` | string | ✅ | 聊天 ID |
| `text` | string | ✅ | 消息文本 |

### 请求示例

```http
POST /v1/chat/send/message
Content-Type: application/json

{
  "chat_id": "ch_abc123",
  "text": "您好，您的订单已发货"
}
```

---

## 4. 获取未读消息数

```
POST /v1/chat/unread
```

获取当前卖家账户的总未读消息数，用于展示消息提醒角标。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `chat_id_list` | array[string] | | 指定聊天 ID 列表（不传则返回所有） |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result` | array[object] | 按聊天分的未读数 |
| `result[].chat_id` | string | 聊天 ID |
| `result[].unread_count` | integer | 未读消息数 |

---

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `chat_id` | string | ✅ | 聊天 ID |

---

## 6. 更新聊天状态

```
POST /v1/chat/update/status
```

更新聊天会话的状态（如关闭/重新打开）。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `chat_id` | string | ✅ | 聊天 ID |
| `status` | string | ✅ | 目标状态：`closed` / `open` |
