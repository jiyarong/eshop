## 1. 获取交易流水列表

```
POST /v3/finance/transaction/list
```

获取卖家账户的交易流水列表（收入、支出、退款等），支持多维度筛选和分页。这是财务模块最核心的接口。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `filter` | object | ✅ | 筛选条件（见下方） |
| `page` | integer | | 页码 |
| `page_size` | integer | | 每页数量 |
| `last_id` | string | | 游标分页 ID |

**filter 子字段：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `date.from` | string (datetime) | 交易起始时间 |
| `date.to` | string (datetime) | 交易截止时间 |
| `operation_type` | array[string] | 操作类型：`Order`、`Return`、`Service`、`Transfer`、`Fee`、`Refund` 等 |
| `posting_number` | string | 货件编号 |
| `transaction_type` | string | 交易方向：`all` / `income` / `expense` |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result.operations` | array[object] | 交易记录列表 |
| `result.operations[].operation_id` | string | 操作 ID |
| `result.operations[].operation_date` | string (datetime) | 操作时间 |
| `result.operations[].operation_type` | string | 操作类型 |
| `result.operations[].posting_number` | string | 货件编号 |
| `result.operations[].amount` | number | 金额 |
| `result.operations[].type` | string | 收支方向 |
| `result.operations[].items` | array[object] | 明细（含 `name`、`price`） |
| `result.operations[].services` | array[object] | 服务费明细 |
| `result.page_count` | integer | 总页数 |

### 请求示例

```http
POST /v3/finance/transaction/list
Content-Type: application/json

{
  "filter": {
    "date": {
      "from": "2024-06-01T00:00:00Z",
      "to": "2024-06-30T23:59:59Z"
    },
    "transaction_type": "all"
  },
  "page": 1,
  "page_size": 50
}
```

### 响应示例

```json
{
  "result": {
    "operations": [
      {
        "operation_id": "1234567890",
        "operation_date": "2024-06-15T12:00:00Z",
        "operation_type": "Order",
        "posting_number": "41517061-0214-1",
        "amount": 5000.00,
        "type": "income",
        "items": [
          { "name": "智能手机壳", "price": 5000.00 }
        ],
        "services": [
          { "name": "平台佣金", "price": 250.00 }
        ]
      }
    ],
    "page_count": 10
  }
}
```

## 2. 获取交易总额

```
POST /v3/finance/transaction/totals
```

获取指定时间范围内的交易汇总（总收入、总支出、净额等），通常放在列表接口前调用作概况展示。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `filter` | object | ✅ | 筛选条件（同交易列表接口的 `filter`） |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result` | array[object] | 汇总数据 |
| `result[].total_income` | number | 总收入 |
| `result[].total_expense` | number | 总支出 |
| `result[].net` | number | 净收入（收入 - 支出） |
| `result[].currency_code` | string | 币种 |

## 3. 获取结算报告列表

```
POST /v2/finance/realization
```

获取 Ozon 的结算报告（Realization Report）列表，显示各周期的款项结算情况。

### 请求参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `date.from` | string (date) | 起始日期 |
| `date.to` | string (date) | 截止日期 |
| `language` | string | 报告语言 |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result.rows` | array[object] | 结算报告行 |
| `result.rows[].realization_id` | string | 结算 ID |
| `result.rows[].date` | string (date) | 结算日期 |
| `result.rows[].amount` | number | 结算金额 |

## 4. 获取费用报告列表

```
POST /v1/finance/accruals
```

获取 Ozon 的费用明细报告——包含平台服务费、物流费、广告费等各类费用扣除明细。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `date` | object | ✅ | 日期范围 |
| `date.from` | string (date) | ✅ | 起始日期 |
| `date.to` | string (date) | ✅ | 截止日期 |
| `page` | integer | | 页码 |
| `page_size` | integer | | 每页数量 |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result.accruals` | array[object] | 费用记录列表 |
| `result.accruals[].operation_id` | string | 操作 ID |
| `result.accruals[].amount` | number | 费用金额 |
| `result.accruals[].type` | string | 费用类型 |
| `result.accruals[].date` | string (date) | 费用产生日期 |

## 5. 获取账户余额

```
POST /v1/finance/balance
```

获取卖家账户当前可用余额和待结算金额。

### 请求参数

无请求体。

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result.balance` | number | 可用余额 |
| `result.reserved` | number | 冻结/预留金额 |
| `result.currency_code` | string | 币种 |
