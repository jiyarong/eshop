# 12 报告管理

> **模块说明**：创建和查询各类数据报告——库存报告、商品报告、销售报告、退货报告等。报告以异步方式生成，通过状态接口轮询获取结果。

共 **7** 个接口。

---

## API 列表

| # | 接口名称 | HTTP 方法 |
|---|---------|----------|
| 1 | [创建报告](#1-创建报告) | `POST /v1/report/create` |
| 2 | [获取报告列表](#2-获取报告列表) | `POST /v1/report/list` |
| 3 | [获取报告详情](#3-获取报告详情) | `POST /v1/report/info` |
| 4 | [获取报告生成状态](#4-获取报告生成状态) | `POST /v1/report/status` |
| 5 | [取消报告生成](#5-取消报告生成) | `POST /v1/report/cancel` |
| 6 | [获取报告模板列表](#6-获取报告模板列表) | `POST /v1/report/template/list` |
| 7 | [创建指定模板的报告](#7-创建指定模板的报告) | `POST /v1/report/template/create` |

---

## 核心流程

```
创建报告 → 轮询状态 → (生成完成) 获取详情下载
                ↓
              取消生成
```

报告类型（`report_type`）：
- `STOCK` — 库存报告
- `PRODUCTS` — 商品报告
- `POSTINGS` — 货件/销售报告
- `RETURNS` — 退货报告
- `FINANCE` — 财务报告
- `SELLER_PRODUCTS` — 卖家商品报告

---

## 1. 创建报告

```
POST /v1/report/create
```

创建一份异步数据报告。报告生成需要一定时间，建议提交后轮询状态接口（每 30 秒一次）。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `report_type` | string | ✅ | 报告类型（见上方类型列表） |
| `params` | object | ✅ | 报告参数（按类型不同） |

**通用 params 子字段：**

| 参数名 | 类型 | 适用类型 | 说明 |
|--------|------|---------|------|
| `date_from` | string (date) | POSTINGS / FINANCE | 起始日期 |
| `date_to` | string (date) | POSTINGS / FINANCE | 截止日期 |
| `language` | string | 全部 | 报告语言 |
| `visibility` | string | 全部 | `ALL` / `VISIBLE` / `ARCHIVED` |

### 响应

```json
{
  "result": { "code": "success" }
}
```

### 请求示例

```http
POST /v1/report/create
Content-Type: application/json

{
  "report_type": "POSTINGS",
  "params": {
    "date_from": "2024-06-01",
    "date_to": "2024-06-30",
    "language": "DEFAULT"
  }
}
```

## 2. 获取报告列表

```
POST /v1/report/list
```

获取当前卖家账户已创建的（或正在生成的）所有报告列表。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `filter` | object | ✅ | 筛选条件 |

**filter 子字段：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `report_type` | string | 报告类型筛选 |
| `status` | string | 报告状态筛选：`WAITING` / `PROCESSING` / `DONE` / `ERROR` |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result` | array[object] | 报告列表 |
| `result[].code` | string | 报告编码（用于查询详情/状态） |
| `result[].report_type` | string | 报告类型 |
| `result[].created_at` | string (datetime) | 创建时间 |
| `result[].status` | string | 当前状态 |

---

## 3. 获取报告详情

```
POST /v1/report/info
```

获取已生成完成的报告的详细数据（包括文件下载链接）。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `code` | string | ✅ | 报告编码 |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `result.code` | string | 报告编码 |
| `result.status` | string | 状态 |
| `result.file` | string | CSV 文件下载链接 |
| `result.created_at` | string (datetime) | 创建时间 |

---

## 4. 获取报告生成状态

```
POST /v1/report/status
```

查询报告是否已生成完成（用于异步轮询）。建议轮询间隔 30 秒。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `code` | string | ✅ | 报告编码 |

### 响应字段

| 字段路径 | 类型 | 说明 |
|---------|------|------|
| `status` | string | `WAITING` → `PROCESSING` → `DONE` / `ERROR` |

---

## 5. 取消报告生成

```
POST /v1/report/cancel
```

取消一个正在排队或处理中的报告生成请求。

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `code` | string | ✅ | 报告编码 |

---

## 6. 获取报告模板列表

```
POST /v1/report/template/list
```

获取可用的报告模板列表（预设的报告类型/参数组合）。
