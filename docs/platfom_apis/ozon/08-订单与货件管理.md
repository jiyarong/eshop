# 08 订单与货件管理

> **模块说明**：管理 FBS / rFBS / FBO 模式的订单与货件全流程——查询、备货、发运、标签打印、取消、配送状态更新、通行证管理等。是 Ozon 卖家 API 中最大、最核心的模块。

共 **45** 个接口，按子模块划分如下。

---

## 核心概念

```
订单(Order) → 货件(Posting) → 备货(Assembly) → 发运(Carriage) → 配送(Delivery)
                                                            ↓
                                                      通行证(Pass)
```

- **Posting（货件）**：订单中的最小发货单元，一个订单可能拆分为多个货件
- **Carriage（发运）**：一次物流运输批次，包含多个货件
- **Pass（通行证）**：车辆进入 Ozon 仓库的入场许可
- **Cutoff**：卖家必须在此截止时间前完成备货

## 一、货件查询

### 1. 获取 FBS 货件列表

```
POST /v1/posting/fbs/list
```

获取 FBS / rFBS 货件列表，支持按状态、时间范围等多维度筛选和分页。

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `filter` | object | ✅ | 筛选条件 |
| `limit` | integer | ✅ | 每页数量 |
| `offset` | integer | | 分页偏移 |

**filter 子字段：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| `since` | string (datetime) | 起始时间 |
| `to` | string (datetime) | 截止时间 |
| `status` | string | 货件状态 |

**响应示例：**

```json
{
  "result": {
    "postings": [{ "posting_number": "...", "status": "awaiting_packaging" }]
  }
}
```

## 二、备货与装运

### 11. 搜集订单（v4 备货）

```
POST /v4/posting/fbs/ship
```

> ⚠️ 状态码 200 不代表备货成功。请使用 `/v3/posting/fbs/get` 检查——如果 `substatus = ship_failed`，需重新备货。

将订单拆分打包，状态变为 `awaiting_deliver`。`packages` 数组中的每个元素为一个包裹。

**需要拆分订单时**：在 `packages` 中传递多个对象，每个对象对应一个独立的包裹。

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `posting_number` | string | ✅ | 货件编号 |
| `packages` | array[object] | ✅ | 包裹列表 |
| `packages[].products` | array[object] | ✅ | 包裹内商品 |
| `packages[].products[].product_id` | integer | ✅ | 商品 ID |
| `packages[].products[].quantity` | integer | ✅ | 数量 |
| `with.additional_data` | boolean | | 是否返回附加信息 |

**请求示例（不拆分）：**

```http
POST /v4/posting/fbs/ship
Content-Type: application/json

{
  "packages": [{
    "products": [
      { "product_id": 185479045, "quantity": 2 }
    ]
  }],
  "posting_number": "89491381-0072-1"
}
```

**请求示例（拆分订单）：**

```json
{
  "packages": [
    { "products": [{ "product_id": 185479045, "quantity": 1 }] },
    { "products": [{ "product_id": 185479045, "quantity": 1 }] }
  ],
  "posting_number": "89491381-0072-1"
}
```

| `products[].product_id` | integer | | 商品 ID |
| `products[].quantity` | integer | | 数量 |
| `products[].exemplarsIds` | array[string] | | 范例 ID（从范例接口获取） |

## 三、发运管理

### 14. 创建发运

```
POST /v1/carriage/create
```

创建第一个 FBS 发运批次。所有状态为"已准备发运"的货件会自动进入该批次。创建后发运状态为 `new`。

**请求参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|:--:|------|
| `delivery_method_id` | integer | ✅ | 配送方式 ID |
| `departure_date` | string (datetime) | | 发运日期（默认为当天） |
| `all_blr_traceable` | boolean | | 是否需包含可追溯商品 |

**响应：**

```json
{ "carriage_id": 123456 }
```

## 附录：货件状态流转

```
awaiting_packaging  →  awaiting_delivery  →  delivering  →  last_mile  →  delivered
(等待备货)              (等待发运)            (运输中)        (最后一英里)   (已送达)
      ↓
  cancelled
  (已取消)
```

| 状态 | 说明 |
|------|------|
| `awaiting_packaging` | 待备货（卖家需拣货打包） |
| `awaiting_delivery` | 已备货，等待发运 |
| `delivering` | 运输中 |
| `last_mile` | 末端配送 |
| `delivered` | 已送达 |
| `cancelled` | 已取消 |
