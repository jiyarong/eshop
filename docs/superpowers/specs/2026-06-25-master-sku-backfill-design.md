# Master SKU 回填脚本设计规格

## 背景

当前 `Ec::Sku` 已支持通过 `master_sku_id` 归属到 `Ec::MasterSku`，但库里仍有一批历史 SKU 尚未建立对应的主产品分组。用户已经提供了待处理 SKU 列表，并确认本次应按“已有基础款编码”归组，而不是为每个 SKU 单独创建主产品。

## 目标

- 为用户提供的这批 SKU 创建缺失的 `Ec::MasterSku`。
- 按明确映射将 SKU 关联到对应 `master_sku`。
- 脚本可重复执行，不重复创建，不覆盖已绑定到其他主产品的 SKU。

## 非目标

- 不尝试自动推断所有编码规则。
- 不修改用户未提供的 SKU。
- 不批量清洗 SKU 名称、俄文名、备注或其他主数据。
- 不通过 migration 或常驻 service 处理这次数据修复。

## 分组规则

- 本次使用脚本内显式映射定义 `master_sku_code -> sku_code 列表`。
- 若某组存在与 `master_sku_code` 同编码的 SKU，则优先使用该 SKU 的 `product_name` / `product_name_ru` 初始化 `Ec::MasterSku`。
- 若不存在同编码 SKU，则使用该组第一个 SKU 的名称字段初始化。
- 对于没有可归并基础款的 SKU，使用自身 `sku_code` 作为 `master_sku_code`，形成单 SKU 主产品。

## 脚本行为

- 文件放在 `script/` 下，沿用仓库现有一次性数据修复脚本风格。
- 逐组处理映射：
  - 查找或创建目标 `Ec::MasterSku`。
  - 查找组内每个 `Ec::Sku`。
  - 若 SKU 不存在，记录缺失并继续。
  - 若 SKU 未绑定 `master_sku`，则关联到目标主产品。
  - 若 SKU 已绑定同一个目标主产品，记为跳过。
  - 若 SKU 已绑定其他主产品，记为冲突并保持原状。
- 脚本结束后输出创建、关联、跳过、冲突、缺失等摘要，便于人工复核。

## 测试

新增一个聚焦模型层行为的测试，覆盖：

- 按映射创建缺失 `Ec::MasterSku` 并绑定多个 SKU。
- 重复执行脚本核心逻辑时不重复创建主产品，也不重复更新已正确绑定的 SKU。
- 已绑定到其他 `master_sku` 的 SKU 保持不变，并被记录为冲突。

## 实现约束

- 变更保持最小：优先新增一个一次性脚本，并抽出尽可能小的可测试 helper。
- 不修改现有 Controller、页面或报表逻辑。
- 测试数据使用唯一 token，避免污染当前非事务测试环境。
